# Audit — 1.4.x hardening pass, 2026-06-13

**Scope**: the **1.4.x surface** — everything added since v1.4.0 that touches
memory, the checkpoint format, or untrusted input: the per-layer mixer **hybrid**
dispatch (`g_layer_kind`/`_lk`), the rung-d **padded uniform-stride** layout
(`_kvw_hyb`, the per-layer weight-init + cache allocation), **checkpoint v6** (the
per-layer-kind region: serialize, load/validate, the padded-np computation), and
the **`--attn-every` CLI**. This is the P(-1) hardening cut that closes the 1.4.x
arc (M14). **Method**: an adversarial multi-agent review — five independent
read-only dimensions, each finding then adversarially verified (refute-by-default)
against the committed code and live repro builds — plus independent edge-case
confirmation. The eighth audit; builds on the M2/M3/M5/M6/M7/M8 and v1.0-final
audits. **Benchmarking is out of scope** (deferred to the dedicated 1.4.6 pass).

## Verdict: GO — 5 go (one after a fix that landed this cut), 0 residual blockers

| dimension | disposition | crux |
|---|---|---|
| Checkpoint-v6 deserialization | **go** | the v6 per-layer region read is bounded BEFORE any `load64` of it (`len < (CKPT_HDR()+pl)*8` → `-1`); `_hybrid_kinds_ok` runs BEFORE `_kvw_hyb`, so the padded-size math never sees an out-of-range kind; the padded `ckpt_expected_np_kvw` reuses the same dim caps (NL≤128, C≤4096, d_c≤C) that bound the product far below i64 overflow; the `-10`/`-11`/`-13`/`-18`/`-46` gates all fire on a forged/truncated v6 image; ZERO allocation before validation. v6 round-trips bit-for-bit (incl. padded mha/ssm) |
| Memory + buffer safety | **go** | the padded K/V region's tail is zero by construction (`t_alloc` zeroes, rosnet.cyr:36-37) and receives no gradient; per-kind caches are allocated for every layer when the kind is present (`_any_kind`), and each dispatch path reads only its own layer's kind's caches; `model_alloc_bytes_hyb` re-derived against the `kinds`-gated allocation (pinned exact by `test_alloc_accounting` for mha/lin, mha/mla, mha/ssm hybrids) |

## Findings → dispositions (1 confirmed; fixed in this cut)

**Fixed in 1.4.5:**
- **CLI stack-buffer overflow (the headline finding).** `--attn-every K` filled the
  fixed 128-slot stack buffer `lkinds[1024]` with `while (hl < cfg_nl) { store64(&lkinds + hl*8, …) }`,
  but `cfg_nl` (from `--layers`, `_atoi`, magnitude-saturating ~1e9) was **not
  bounded** before the loop — the NL≤128 cap lives in `model_config_ok_moe`, reached
  only inside `model_init_*`, which runs AFTER the loop. So `attn11 --layers N
  --attn-every K` with N>128 (no `--load`, learned-abs) wrote 8·N bytes past the
  1024-byte stack buffer. **Confirmed live**: `--layers 100000 --attn-every 2` →
  **SIGSEGV (exit 139)** on the committed binary; the control `--layers 100000`
  *without* `--attn-every` is a clean `model_config_ok` rejection — isolating the
  hybrid write loop as the sole trigger. Same class as the documented M7 `--layers`
  heap-OOB. **Fix**: an `if ((cfg_nl < 1) || (cfg_nl > 128))` guard before the loop
  (main.cyr), capping `cfg_nl` to the `lkinds` capacity (= `model_config_ok`'s NL
  cap). After the fix the same exploit exits 1 cleanly. Independently surfaced by
  two of the five review dimensions; both verifiers reproduced the SIGSEGV and
  confirmed the guard. **Regression**: a new `make smoke` target (in the `release`
  gate) asserts hostile `--layers/--attn-every` combos reject cleanly (exit < 128)
  and a valid hybrid still builds — the grad-check/fuzz harnesses exercise the math
  and the checkpoint loader but NOT the CLI arg path, which this closes.

**Refuted (raised, then verified NOT real):** none — the other four dimensions and
the verification pass returned no further confirmed findings (`refuted: []`).

**Accepted residuals (notes, no action — safe by construction):**
- The `lkinds[1024]` buffer size, the new CLI guard's `128`, and
  `model_config_ok_moe`'s NL≤128 cap are three copies of one constant (`128 i64 =
  1024 bytes`). They cannot drift silently today (the guard comment ties them), but
  a future NL-cap change must update all three; a shared `NL_MAX` constant across
  `main.cyr`/`model.cyr` would make the coupling explicit. Deferred (hot config
  path; out of scope for a one-change hardening cut).
- The rung-d padding wastes a few % params on smaller-kind layers in a mixed-layout
  hybrid (X013: +1 440 / ~4%). Accepted by design (ADR 0012) — the price of a
  uniform stride without a per-layer-offset rewrite; `{mha,gqa,lin}` hybrids are
  exact.
- **CLI-argument fuzzing is a harness gap.** The fuzz harness mutates checkpoint and
  BPE images, not CLI args; the new `smoke` target is a fixed-vector regression, not
  a fuzzer. A property-style CLI-arg fuzzer is a reasonable future hardening item
  (would have caught this finding generatively).

## Method note

Five read-only review dimensions (checkpoint-v6 security, memory/buffer, hybrid
correctness, frozen surface, edge/robustness) ran in parallel; each finding was
then handed to an independent adversarial verifier (instructed to refute unless the
code confirms) before any fix. 2 raw findings, both the same overflow, both
verified genuine; the fix + the `make smoke` regression landed this cut. Foundational
assumptions checked directly: `t_alloc` zeroes (rosnet.cyr:36-37, underpins the
padded-tail-is-zero argument), and the uniform/v5 paths are byte-identical.
