## Security checklist (first-party standards)

- **Input validation** — checkpoint headers validate magic/version/every config
  field/recomputed param count/exact size *before any allocation*; corpus loads
  are size-capped via `fstat`. ✅
- **Buffer safety** — all training buffers are allocated once at fixed,
  dimension-derived sizes; no per-step allocation (verified by the soak test's
  `alloc_used()` invariant). SIMD scratch buffers are fixed 32-byte (4-lane). ✅
- **Syscall review** — file `read`/`write` loop over short transfers; `fsync`
  failure aborts the atomic save; negative errnos propagate (not swallowed as
  EOF). ✅
- **Pointer/bounds validation** — no untrusted value indexes memory before its
  range is checked; the `INT64_MIN` sampler edge (negate-stays-negative → OOB)
  was fixed with a sign-bit mask. ✅
- **No command injection** — no `sys_system`/`exec`; CI security scan greps for
  `syscall(59)`/`sys_system` and is clean. ✅
- **No path traversal / symlink** — every untrusted open uses `O_NOFOLLOW`
  (corpus read, checkpoint read, checkpoint temp write). ✅

## M2 — data & persistence (fixed in 0.3.0)

| # | Sev | Finding | Fix / guard |
|---|-----|---------|-------------|
| 1 | **HIGH** | Resume with a corpus whose byte-vocab differs from the checkpoint's silently mis-indexed `g_data` against the restored embeddings (only vocab *size* was checked). | Hard-reject (`-15`) on any vocab mismatch; tracks the corpus vocab size separately (`g_corpus_vlen`). Regression: `test_ckpt_vocab_mismatch`. (Also surfaced a real bug: `corpus_set` never set `g_V`.) |
| 2 | LOW | Checkpoint `dropout` field used unvalidated (NaN / `≥1.0` poison training math). | Bit-pattern validation to finite `[0,1)` → `-14`. Regression: `test_ckpt_reject` (NaN + 1.0 cases). |
| 3 | LOW | Mid-stream `read` error treated as clean EOF in `secure_read_file`/`ckpt_load_file`. | Propagate negative errno; only `n==0` is EOF. |
| 4 | LOW | Oversize stdin corpus silently truncated (inconsistent with the file path). | Read `cap+1`, reject `> cap` with `-2`. |
| 5 | LOW | Non-atomic checkpoint save (`O_TRUNC`, no fsync). | Deferred to M4 → crash-atomic save (temp+fsync+rename). |
| 6 | LOW | `INT64_MIN` from the PRNG defeats the abs() in `sample_window` → negative modulo → OOB corpus read. | Mask the sign bit instead of negating. |
| 7 | LOW | Resume-determinism test didn't exercise the dropout PRNG draw schedule. | Added a `dropout=0.1` determinism case (`test_resume_dropout`). |
| 8 | LOW | Generation runs on an untrained/short-corpus model with no notice. | Print "model is untrained" when `g_step == 0`. |

## M3 — SIMD matmul/attention (fixed in 0.4.0)

| # | Sev | Finding | Fix / guard |
|---|-----|---------|-------------|
| 1 | **MEDIUM** | The production attention head dim (`hd=8`, two SIMD chunks) was never grad-checked — only `hd=6` (one chunk) and `hd=2` (scalar). A second-chunk bug could hide. | Added `hd=8` and `hd=10` attention grad checks (`_attn_check` at C=16/20). |
| 2 | LOW | Comments claimed "bit-identical" without the toolchain caveat. | Reworded; added `test_simd_contract` pinning `f64v_fmadd == scalar mul+add`. |
| 3 | LOW | `f64v_fmadd` lowers to `mulpd`+`addpd` (two roundings) on x86 — bit-identity is toolchain-dependent and untested. | `test_simd_contract` pins it (and is arch-aware: bit-exact x86 / within-rounding aarch64). |
| 4 | INFO | The `dx`/score *dot* paths use a 4-lane tree reduction → not strictly bit-identical to a sequential scalar dot. | Docs corrected: AXPY paths bit-identical, dot paths within rounding (~1e-16). |

## M4 — portability & robustness (fixed in 0.5.0)

| # | Sev | Finding | Fix / guard |
|---|-----|---------|-------------|
| 1 | **MEDIUM** | `secure_write_atomic` ignored the `fsync` return — a failed flush still renamed over the good checkpoint, defeating the crash-atomic guarantee. | Abort on `fsync < 0` (close + unlink the temp, leave the target intact, return `-2`). |
| 2 | **MEDIUM** | Attention grad-check tolerance was loosened to 1e-4 on *all* arches; x86 (native `exp`) lost detection headroom. | Arch-aware: x86 keeps 1e-5, aarch64 relaxes to 1e-4 (exp-polyfill finite-difference noise). |
| 3 | LOW | A failed/short write left an orphaned `.tmp` sidecar. | `sys_unlink` the temp on the error path. |
| 4 | LOW | `test_nan_guard` only asserted `g_step`, not the train return or weight pristineness. | Now asserts `train() == -1` and that a finite weight is unchanged after the guard fires. |
| 5 | INFO | Temp-path buffer size / `.tmp` encoding. | Verified correct; no change. |

## Carried forward

- Crash-atomic save fsyncs the file but not the parent directory; on a crash
  immediately after `rename`, a filesystem may surface the *prior* checkpoint
  (a safe failure — never a corrupt file). Parent-dir fsync is a possible future
  hardening.
- Checkpoints are native-endian and not cross-architecture portable (see
  ADR 0004); loading an untrusted/foreign checkpoint is out of the threat model.
